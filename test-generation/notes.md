I initially took a while before deciding on a website. At first, I thought I should design my own website integrating the login, dashboard and api features but it would take too long given the time frame, so I scratched that idea and went ahead with one of the websites online designed for testing purposes with api documentation also present. That is, https://practice.expandtesting.com/notes/app.
The login module had all the basic features like login, register and forgot password but it didn't have features such as session-expiry and brute force lockout so I decided to write the test cases for it and not implement it in playwright.
For the dashboard, I decided to use a specific account with existing notes to validate the different tests such as widget loading, data accuracy ,filter/sort behaviour etc.
For the API layer, my initial prompt didnt work as the api documentation page online was protected from any kind of automation efforts so I had to download the page and ask the llm to generate test cases with reference to that, there were a few failiures/different outcomes due to that but I cross verified and validated the outcomes.
